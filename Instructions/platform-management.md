# Platform Management (Overview)

## What the client gets
One social platform, fully run by Nova. The client picks the platform — Nova handles content production, scheduling, community, and reporting on that single channel. This is **one service**, not five. The deliverables shape depends on the platform chosen.

If a worker is briefed "Platform Management" without a specific platform named, start here. Confirm the platform with the account manager, then jump to the relevant per-platform instruction file.

## The picker

The client chooses ONE platform:

1. **Instagram** — see `instagram-management.md`. Feed posts, Reels, Stories, DMs, hashtag strategy.
2. **TikTok** — see `tiktok-management.md`. Short-form video, FYP hook engineering, trending-sound strategy, comment seeding.
3. **YouTube Shorts** — see `youtube-shorts.md`. Shorts-only (60s vertical). No long-form. No chapters, no end screens.
4. **Twitter / X** — see `social-media-management.md` for the X-specific section (or request a dedicated `twitter-management.md` when scope grows).
5. **LinkedIn** — see `linkedin-growth.md`. Personal-brand or company-page posting cadence + comment-on-others strategy.

IG, TikTok, and YouTube Shorts share the same deliverable shape: posts + Stories/community + short-form video, scheduled and managed. Twitter/X and LinkedIn have their own deliverable shapes — text-first, with different community rhythms.

## Pricing
**AED 1,200–2,000/mo** depending on platform, cadence, and complexity.

- Lower end (AED 1,200): one platform, base cadence (e.g. 3 posts/week + light community).
- Upper end (AED 2,000): one platform, higher cadence (e.g. 5 posts/week + daily Stories + active community seeding).

This is a published price band. The website lists the band, not five separate per-platform numbers. The exact figure is set in the proposal.

## What Worker does — step by step

1. **Confirm the platform.** Account manager passes the platform name in the brief. If missing, ask before doing anything else.
2. **Open the per-platform instruction file** from the picker above. That file is the working SOP — follow it.
3. **Standard kickoff, every platform:**
   - Account audit (see per-platform file for the exact audit prompt)
   - Create a `PLAYBOOK.md` in the client's Drive folder (IG → `IG_PLAYBOOK.md`, TikTok → `TIKTOK_PLAYBOOK.md`, etc.)
   - Lock the content pillars (usually 5)
   - Lock the cadence based on tier of this engagement
4. **Run the monthly cycle** per the per-platform file: ideation, production, scheduling, community, report.
5. **Cross-reference shared files** when needed:
   - `ugc-short-form-video-ads.md` for short-form video production (used by IG Reels, TikTok, YouTube Shorts)
   - `social-media-management.md` for DM + comment standard ops
   - `daily-content-engine.md` if the client is also buying the lifestyle-feed add-on

## Tools used
- Claude.ai Project — content, captions, community, reports (per-platform prompts live in the per-platform files)
- Platform-native scheduler when it exists (TikTok Studio, YouTube Studio) — beats third-party for algorithmic weight
- Metricool or Later — cross-platform calendar view (optional)
- Per-platform analytics dashboard

## Time required
- Kickoff: 2-3 hours one-time per platform
- Monthly ops: 16-40 hours depending on platform + cadence (see per-platform file for the exact band)

## What to send the client
- `PLAYBOOK.md` link on kickoff
- Monthly content calendar (Google Doc) on day 1 of the month
- Scheduled posts visible in the client's account
- Monthly report (PDF or Google Doc) with the platform's key metrics

## Quality check - CTO & COO review
- The right platform is being managed (the one the client chose, not a platform the worker prefers)
- Cadence matches the proposal — no silent drift up or down
- Voice matches the playbook — no generic Claude voice
- Community engagement is happening on cadence — not just publishing
- Monthly report is delivered by the 3rd of the following month

## Tier availability
Standalone add-on. Not Nova-tier-locked — any tier client can add Platform Management for one platform. If a client wants multi-platform coverage, sell `social-media-management.md` instead (multi-platform service).
